Level 0 Design

Introduction:

1. Name your project.
2. Name the level 0 module.
3. Name the input(s) to the level 0 module and provide a brief description of each input.
4. Name the output(s) of the level 0 module and provide a brief description of each input.
5. Describe the functionality of the module. Click the “Add Input” / “Add Output” buttons to include additional inputs / outputs as appropriate to your level 0 design.

When your level 0 design is complete, click the ‘Design Complete’ button and a block diagram and associated table for your design will be provided along with additional questions for you to consider regarding your level 0 module.






     



     




Feedback on level 0 design: A common error in creating functional decomposition diagrams and tables is to fail to be consistent in naming signals. For the level 0 diagram this app has taken the names you provided for the input(s) and output(s) and used them both in the diagram and in the table to ensure consistency. As you proceed to level 1 and level 2 designs, use the names you provided in the level 0 design when the corresponding signals are found in other levels of the design. A second common error in design is to fail to consider that all inputs and outputs must be measurable which will become obvious during the testing and verification phase of the design process. Did you consider testability when choosing signals? If not, edit your design. If you are comfortable with your level 0 design, select the print pdf option to create an image of your level 0 design.
